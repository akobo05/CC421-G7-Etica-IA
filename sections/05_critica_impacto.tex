\section{Crítica y evaluación de impacto}
La responsabilidad es tratar de definir qué considera apropiado el programador de software/científico, pero para Kant esta posibilidad de proponer una decisión de auto autónomo debe ser un acto moral según el imperativo categórico; en contraste, para Bentham estaría de acuerdo en utilizar un modelo matemático para definir un nivel de moralidad, no se negaría como Kant al decidir si el sistema comete un acto menos perjudicial. Mill posiblemente establecería un modelo donde el auto autónomo calcule en tiempo real cuáles son las implicaciones de una decisión, como qué tan responsables son los involucrados.

\medskip

En síntesis, bajo este marco teórico, el ingeniero de software/científico de datos no tiene responsabilidad meramente técnica, sino profundamente ética en la defiinición de las condiciones bajo las cuales actuará el sistema e interactuará con el usuario. En ese sentido, podemos dividir la responsabilidad del ingeniero en 3 dimensiones basadas en los filósofos analizados:   
\begin{itemize}
    \item Responsabilidad desde Kant (''educativo''):
    El ingeniero / científico de datos debe garantizar la autonomía del usuario mediante la transparencia de las limitaciones del sistema, para que el usuario pueda tomar decisiones informadas y no se vuelva un medio de la tecnología. Esto implica diseñorar un sistema que no genere ''exceso de confianza''. 
    \item Responsabilidad desde Bentham (''epistémico''):
    El ingeniero / científico de datos es responsable de la calidad de los datos y también del sesgo que estos tendrán. Ignorar que el sistema de sensores no detecta ciertos escenarios, como por ejemplo la aproximación de un tráiler, es una falla en el cálculo de ''certeza utilitarista''.

    \item Responsabilidad desde Mill (''normativo'')
    El ingeniero / científico de datos debe seguir principios secundarios que podrían ser los estándares de seguridad industrial, por ejemplo. Estos actuarían como ''reglas de utilidad'' que protegerán la sociedad a largo plazo, evitando que en próximos trabajos se tenga que reinventar la ética en cada línea de código.
\end{itemize}
